# Linux学習記録 Day16
## アクセスログ解析とトップアクティブIPの自動特定

- 昨日（Day15）習得したデータのあぶり出しを応用。
- 大量のアクセスログから最もアクセス（負荷）をかけている犯人のIPアドレス1件をピンポイントで全自動特定する実戦。

## 覚えたサブコマンド・オプション
- **sort -rn** 
  - `-n`文字ではなく数値の大きさで並び替える
  - `-r`reverse（逆順）。数値を**大きい順番,降順**に並び替えて最上部にトップを持ってくる
- **head -n 1**
  - `head`：データの頭（最初）から切り出すコマンド
  - `-n 1`：**「最初の1行だけ」**に絞り込む。2位以下の余計なデータを削ぎ落とすプロの引き算テクニック

## 構築したパイプライン
```bash
cat access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head -n 1
```

## 実行結果
```text
   3 192.168.1.10
```
## 間違えたこと
- **実行時にコマンド誤認識のエラー（`command not found: -c`）**
  - **原因**`| sort | uniq | -c` や `| uniq | -c | access.log` のように、オプションであるはずの `-c` の手前に不要なパイプ（`|`）を挟んで記述。システムが `-c` を独立した存在しないコマンドとして認識。
  - **解決**オプションは直前のコマンドのすぐ後ろにスペースを空けて記述する必要があると理解し、不要なパイプを削除して `uniq -c` の形に修正。

- **実行時に `awk` の構文エラー（`usage: awk ...`）**
  - **原因**単に `cat access.log | awk | sort ...` と実行。ハサミの役割である `awk` に対して「何列目を切り抜くか」という肝心の命令（引数）を渡し忘れていたため。
  - **解決**1列目のIPアドレスをピンポイントで抽出するため、引数として `'{print $1}'` をしっかり指定する形に修正し、エラーなしで集計表の出力を成功。
