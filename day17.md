# Linux学習記録 Day17
## 複数スクリプトのCron同時運用における連鎖停止障害のトラブルシューティング

- 新しくディスク容量自動監視スクリプトをcronに追加した際、既存の定期自動バックアップまで巻き添えで実行されなくなった。環境依存によるバグをデバッグし、双方の自動化システムを安全に両立させた。

## 構築した環境
## `crontab -l`（Cron設定）
```text
0 9 * * * /Users/r.h/webusaitozidouka/backup.sh
0 * * * * /Users/r.h/disk_monitor.sh
```

## `/Users/r.h/disk_monitor.sh`（修正後の完全版）
```bash
#!/bin/bash
## セキュリティのため実際のWebhook URLは隠しています
WEBHOOK_URL="https://slack.com"
THRESHOLD=80

export PATH=/usr/bin:/bin:/usr/sbin:/sbin

# 【修正】お尻の余計なクォーテーションを消して、綺麗に閉じます
USAGE=$(df -h / | grep -o '[0-9]*%' | tr -d '%')

USAGE=${USAGE:-0}

if [ "$USAGE" -gt "$THRESHOLD" ]; then
    PAYLOAD="{\"text\":\"⚠️ディスク残量警告 現在の使用率: ${USAGE}%\"}"
    /usr/bin/curl -X POST -H "Content-type: application/json" --data "$PAYLOAD" "$WEBHOOK_URL"
fi
```
## 間違えたこと

- 新機能（容量監視）を追記した際、環境や構文の複数の罠に抵触し、システム全体がロックされた。

- **実行時に設定ファイル（crontab）の分断バグ**
  - **原因**：1行目と2行目の設定の間に不要な完全な空行を挟んで記述。cronは空行を見つけた時点で以降の読み込みを終了する仕様。
  - **解決**：`crontab -e` から間の空行を削除して上下に密着させ、末尾（ファイルの最終行）にのみMacの実行に必須となる改行を挿入する形に修正。

- **Cron実行時におけるコマンドのパス（PATH）未認識エラー**
  - **原因**：手動のターミナルとは異なり、cronの実行ロボットは環境変数（PATH）を持っていない。スクリプト内の `curl` などのコマンドがどこにあるか分からず内部でエラー
  - **解決**：スクリプトの先頭付近に `export PATH=/usr/bin:/bin:/usr/sbin:/sbin` を追記し、コマンドの探索ルートを明示的に認識。

- **MacのCron依存による列ズレと、構文のタイポ（クォーテーション重複）**
  - **原因**：`df -h` の出力を `awk` の列番号で切り抜こうとした際、cron経由だと出力の列順がズレて結果が空っぽになり、if文の数値比較でパニック。さらに、それを修正する過程で末尾のシングルクォーテーションが重複する記述ミスが混ざり、再度フリーズ。
  - **解決**：列番号に依存しない `grep -o '[0-9]*%'` によるパーセント文字の直接抽出に切り替え、タイポを完全に排除。さらに、値が空の際の安全装置（`${USAGE:-0}`）を組み込むことで、連鎖停止を防ぐ。
## 気づいたこと
- 1日に何回もバックアップした場合、同じファイルに保存される。それを知らずに、正しく動くか試しにバックアップした時、新しいファイルない＝バックアップできていないと勘違いして何度もバックアップしていた。
- これからは1日に何度もバックアップする時は、ファイルの中もしっかり見る。
