## 今日の成果（完成したもの）
- Docker上で動作するディスク監視システム（Slack通知 + cron + 状態管理）を実装し、簡易SRE監視構成を再現した

最終的に以下の仕組みが完成した

- 80%を超えたらSlackへ警告送信

### ディスク使用率の取得

- Dockerコンテナ内の/data領域の使用率を取得し、数値化して判定できるようにした。
### Slack Webhookによる通知
```bash
curl -X POST -H "Content-type: application/json" \
--data '{"text":"test message"}' YOUR_WEBHOOK_URL
```
- Slack Incoming Webhookを使用して、外部からメッセージ送信できることを確認した。
### 3. ディスク監視スクリプト作成（disk_monitor.sh）
- 以下の「簡易SRE監視システム」を構築した
```bash
#!/bin/bash

WEBHOOK_URL="https://hooks.slack.com/services/XXXX"
THRESHOLD=80

export PATH=/usr/bin:/bin:/usr/sbin:/sbin

USAGE=$(df -h /data | awk 'NR==2 {print $5}' | tr -d '%')

STATE_FILE="/tmp/disk_alert"

if [ "$USAGE" -gt "$THRESHOLD" ]; then
    if [ ! -f "$STATE_FILE" ]; then
        echo "1" > "$STATE_FILE"

        PAYLOAD="{\"text\":\"⚠️ ディスク警告: ${USAGE}% (/data)\"}"

        curl -X POST -H "Content-type: application/json" \
        --data "$PAYLOAD" "$WEBHOOK_URL"
    fi
else
    rm -f "$STATE_FILE"
fi
```
### cronによる自動実行
```bash
crontab -e
```
```bash
*/5 * * * * /data/disk_monitor.sh >> /data/monitor.log 2>&1
```
- 5分ごとに自動で監視が走るように設定した。

## 学んだこと
- 監視システムは「取得・判定・通知・状態管理」で構成される
- 一度通知するだけではなく「状態を持つ設計」が重要
- Dockerとホストのディスクは別物ではなくマウント関係で繋がっている場合がある
- `cron`を使うことで定期実行が簡単に実現できる
- SRE的な考え方では「異常検知 + 自動通知 + 再通知制御」が基本になる
- シェルスクリプト・cron・外部API連携を組み合わせて「自動化された監視処理」を設計・実装できるようになった
