# ops-lab CLIの機能改善
## 実践したこと

ops-labのCLIに新しい機能を追加し、使いやすさを改善。

まず、バージョンを確認するための`version`コマンドを追加。

新しく`cli/version.sh`を作成し、`ops_version()`関数を実装。

さらに、`ops_cli.sh`から`version.sh`を読み込むように変更し、`ops`ファイルの`case`文にも`version`を追加。

これにより

000
ops version
000

でCLIのバージョンを表示できるようになった。

また、`help.sh`も更新し、新しく追加したコマンドを含めて、現在利用できるコマンド一覧が表示されるように整理した。

次に、存在しないコマンドを入力した場合の表示を改善した。

今までは

000
Unknown command: aaa
Run 'ops help' for usage.
000

と表示するだけだったが、コマンド一覧を配列で管理し、

000
COMMANDS=(
  help
  version
  back
  up
  ...
)
000

として管理する構成へ変更。

Unknown command時には、この配列から利用可能なコマンド一覧を表示するようにした。

さらに、入力された文字列がコマンド名の先頭と一致した場合は、

000
ops sta
000

のような入力に対して、

000
Unknown command: sta

Did you mean:
  status
000

と候補を表示する機能を追加。

CLIの利便性が向上し、入力ミスをした場合でも正しいコマンドを見つけやすくなった。

## 間違えたこと

- 最初は候補表示を実装したつもりだったが、完全一致で判定していたため、`ops sta`のような入力では候補が表示されなかった。

- 原因は

000
[[ "$cmd" == "$1" ]]
000

としていたためで、これでは完全一致しか判定できなかった。

- 前方一致へ変更することで、入力途中でも候補を表示できるようになった。

## 学んだこと

- CLIはコマンドを実行できるだけでなく、入力ミスをサポートすることで使いやすさが大きく向上することを学んだ。

- コマンド一覧を配列で一元管理すると、新しいコマンドを追加する際の管理がしやすくなることを理解した。

- 完全一致と前方一致では動作が異なり、用途に応じて比較方法を選ぶことが重要だと学んだ。
