# Linux学習記録 Day50  
# Memory MonitorとProcess Monitorを実装して監視対象を拡充
## 実践したこと  
  
- `memory_monitor.sh` を新たに作成し、`free` コマンドを利用してメモリ使用率を取得できるようにした。  
- メモリ使用率に応じて `OK`・`WARN`・`CRITICAL` を判定し、ログ・メトリクス・状態ファイルを更新する機能を実装した。  
- メモリ情報の取得に失敗した場合のエラーハンドリングを追加し、異常時でも適切に終了できるよう改善した。  
- `process_monitor.sh` を新たに作成し、`pgrep` を利用して指定したプロセス（`cron`）の稼働状況を監視できるようにした。  
- プロセスの状態に応じて `OK`・`CRITICAL` を判定し、状態変化時のみSlackへ通知する仕組みを実装した。  
- 監視対象のプロセス名を `PROCESS_NAME="${PROCESS_NAME:-cron}"` とし、環境変数から変更できるよう改善した。  
- `process.prom` に監視結果を出力し、メトリクスとして利用できるようにした。  
- `bash -n` による構文チェックと `ShellCheck` による静的解析を実施した。  
- Dockerコンテナ内で `memory_monitor.sh` と `process_monitor.sh` を実行し、ログ・メトリクス・状態ファイルが正常に生成・更新されることを確認した。  
  
## 間違えたこと  
  
- `process_monitor.sh` を作成する際、`memory_monitor.sh` をコピーしたため、不要な処理や変数が残ってしまった。  
- `if` を誤って `cif` と入力し、構文エラーを発生させてしまった。  
- 使用しなくなった `STATUS` 変数を残したままにしてしまい、`ShellCheck` の警告が表示された。  
- Dockerコンテナ内で `docker` ディレクトリへ移動せずにスクリプトを実行し、ファイルが見つからないエラーになった。  
  
## 覚えたこと・コマンド  
  
- `free`  
  - Linuxでメモリ使用状況を確認できる。  
  
- `pgrep -x プロセス名`  
  - 指定したプロセスが動作しているか判定できる。  
  
- `PROCESS_NAME="${PROCESS_NAME:-cron}"`  
  - 環境変数が指定されていればその値を使い、未指定ならデフォルト値を利用できる。  
  
- `bash -n process_monitor.sh`  
  - シェルスクリプトの構文チェックを実行できる。  
  
- `shellcheck process_monitor.sh`  
  - シェルスクリプトの静的解析を実行できる。  
  
## 学んだこと  
  
- CPU・メモリ・ディスクだけでなく、プロセスの稼働状況も監視することで、より実用的な監視システムを構築できる。  
- スクリプトをコピーして作成する場合は、不要な変数や処理が残っていないか確認することが重要である。  
- `ShellCheck` は不要な変数や記述漏れなど、品質向上につながる問題を見つけるのに役立つ。  
- 監視スクリプトは、ログ・メトリクス・状態ファイル・通知まで一連の流れを実装することで、運用しやすい構成になる。  
- 実装後は、構文チェック・静的解析・Dockerコンテナ内での動作確認まで行うことで、正常に動作することを確実に確認できる。
