# Linux学習記録 Day52
# バックアップ機能の拡張と世代管理改善
## 実践内容

- バックアップスクリプトを改善し、複数ファイルをバックアップできるように変更した。
- バックアップ対象を配列（`TARGETS`）で管理する設計へ変更した。
- `index.html` に加えて `docker-compose.yml` もバックアップ対象に追加した。
- バックアップ処理をループ化し、対象ファイルごとにバックアップファイル名を自動生成するようにした。
- 世代管理もファイルごとに行うよう修正し、それぞれ独立してローテーションされるよう改善した。
- `ops backup-list` の表示を改善し、ファイル名・サイズ・更新日時・保存数を確認できるようにした。
- Dockerコンテナ内でバックアップスクリプトを手動実行し、`index.html` と `docker-compose.yml` のバックアップが正常に作成されることを確認した。
- cron設定やDockerボリュームを確認し、自動バックアップの動作や保存先について調査した。

## 間違えたこと

- Dockerのボリュームマウントの仕組みを十分理解できておらず、コンテナ内だけにバックアップが保存されていると勘違いしていた。
- ログにバックアップ成功と表示されていても、実際の保存先やファイルの状態を確認していなかった。
- MacとDocker(Linux)で同じBashが使われていると思っていたが、`mapfile` がMac標準Bashでは使えないことを後から知った。

## 学んだこと

- バックアップ対象を配列で管理すると、対象ファイルを追加するだけで簡単に拡張できる。
- ファイルごとに世代管理を行うことで、種類ごとのバックアップを安全に管理できる。
- Dockerのボリュームマウントによって、コンテナとMac側で同じディレクトリを共有できる。
- LinuxとMacではBashのバージョンが異なるため、使えるコマンドに違いがあることを理解した。
- ログだけではなく、実際に保存されたファイルまで確認することが重要だと学んだ。

## 覚えたこと・コマンド

```bash
ops backup-list
```
バックアップ一覧を表示する。
```bash
ops cron
```
cronの設定内容を確認する。
```bash
ops exec crontab -l
```
コンテナ内のcrontabを確認する。
```bash
ops exec ps aux | grep cron
```
cronプロセスが動作しているか確認する。
```bash
ops exec date
```
コンテナ内の現在時刻を確認する。
```bash
ls backup
```
バックアップディレクトリの内容を確認する。
```bash
tail -30 logs/backup.log
```
バックアップログの最新30行を表示する。
