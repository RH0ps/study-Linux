# Linux学習記録 Day15
## 覚えたコマンド
- `find`ファイルやフォルダを検索するコマンド
- `-name`名前を指定して検索するオプション
- `"*fruits*"`前後に文字がついてもいいから「fruits」が含まれるファイル
- `2>/dev/null`エラー画面への表示を非表示にする
- `awk`文章を編集・抽出するコマンド
- `'{print $1}'`1列目の文字だけを抽出して画面に表示する
- `sort`データをアルファベット順などに並び替える
- `uniq`上下に連続した重複行を1つにまとめる
- `-c`重複が何個あったか回数を数える

## 探す系コマンドの「3大使い分け」
- **ファイル自体**を探す探偵は **`find`**
- ファイルの**中身の文字**を探す虫眼鏡は **`grep`**
- **コマンドの正体**を探すナビゲーターは **`which`**

## 詰まったこと
- コマンドいっぱいありすぎて覚えるの大変

## 間違えたこと
- 最初に `cat fruits.txt` を含む複合コマンドを実行した際、まだファイルが存在しなかったため、エラー
`cat: fruits.txt: No such file or directory`
- 解決 重複の中に1つだけ孤立したデータ（ぶどう）が混ざった果物リストを作成

```bash
cat << 'EOF' > fruits.txt
りんご
みかん
りんご
ぶどう
りんご
みかん
EOF
```

## コマンドによる犯人（1回しかいないデータ）のあぶり出し
- データ作成後`find`でのファイル検出を確認し、以下のコマンドを実行

```bash
cat fruits.txt | awk '{print \$1}' | sort | uniq -c
```

## 実行結果
```text
   1 ぶどう
   2 みかん
   3 りんご
```
